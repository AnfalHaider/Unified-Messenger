1.2 Sidebar & Instance Layout State
-----------------------------------
- Model: MessengerInstance (Id, DisplayName, ProfileName, StartUrl, Platform,
  branding, WorkspaceCategory Personal/Professional, SortOrder, mute, memory tier)
- Persistence: InstanceRegistryService → %LocalAppData%\UnifiedMessenger\instances.json
- UI: WorkspaceSidebar.Refresh() builds Overview / Pro / Business / Personal sections
- Features: badges, connection status, ProfileAvatarService avatars, drag-drop reorder
- Compact sidebar: partial (56px rail via MainWindowShellLayout + SetCompactDisplay)

1.3 Core Dashboard Telemetry Status
-----------------------------------
Professional tab (real data binding):
- KPIs: MessageAnalyticsService.CaptureProfessionalSnapshot() → analytics.json
- WeeklyActivityChart: 7-day sent/received from analytics
- Urgency triage: MessageTriageService (in-memory, lexicon scorer, urgency >= 30)
- SentimentActivityChart: triage WeeklySentiment buckets
- Google reviews / Meta panels: ProfessionalWorkspaceService + WebView scrapers
- Branch filter: DashboardPageHelper filters professional instances
- Export: JSON/CSV analytics export on Professional tab

Personal tab (real data binding):
- Recent activity: NotificationHub (personal instances only)
- Resource tiles: ResourceMonitorService + InstanceSessionManager + hub (30s refresh)
- Global search over instances and activity

XAML placeholders: many TextBlocks default to "—" or "0" until Refresh* runs.

1.4 WebView2 Integration Layer
------------------------------
- WebViewProfileManager: shared CoreWebView2Environment, per-profile WebView2
- InstanceSessionManager: one WebView per instance, suspend/LRU, WebMessageReceived
- PlatformAdapterFactory: WhatsApp, Telegram, Messenger, Slack, Discord, Signal,
  Teams, Meta Business, Google Business, Generic
- Scripts injected at document created: adapter-core, connection-handshake, platform JS,
  optional ai-draft-inject + inbound-message-monitor (Meta/Google)
- WebMessage types: badge-count, notification-preview, adapter-ready, heartbeat,
  connection-status, message-sent, meta-inbound-message, google-review-*,
  inbound-message-selected
- Auto-draft path: inbound selection → triage enqueue + AutoDraftOrchestrator →
  Ollama → WebViewDraftInjector (no auto-send in ai-draft-inject.js)

1.5 Local AI & Process Management
---------------------------------
- OllamaHttpClient: http://127.0.0.1:11434 (api/tags, generate, pull)
- OllamaBootstrapService: process start when auto-bootstrap enabled
- OllamaOrchestrationService: warmup on launch, streaming generate, model pull
- AutoDraftOrchestrator: gated on EnableLocalAi + EnableAutoDraft
- LocalAISettingsPage + ollama-models.json, ai-draft-prompts.json
- Message triage does NOT use Ollama (lexicon-only MessageTriageScorer)
